# DimaTech Assignment Task

Асинхронное REST API приложение на Sanic с PostgreSQL.

## Стек

- **Python 3.10**
- **Sanic**
- **SQLAlchemy 2.0** (async)
- **PostgreSQL**
- **Alembic** (для миграций)
- **Argon2** (для хеширования паролей)
- **python-jose** (JWT токены)
- **Docker / Docker Compose**

---

## Запуск с Docker Compose

### 1. Клонируем репозиторий

```bash
git clone https://github.com/nick-redwill/dimatech-assignment-task
cd dimatech-assignment-task
```

### 2. Создаем `.env` файл из шаблона (значения по умолчанию для докера уже прописаны)

```bash
cp .env.template .env
```

### 3. Запускаем

```bash
docker compose up --build
```

Приложение будет доступно по адресу: `http://localhost:8000`

При запуске автоматически выполняются миграции и создаются тестовые пользователи.

---

## Запуск без Docker

### 1. Устанавливаем зависимости

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Создаем `.env` файл из шаблона и заполняем актуальными значениями

```bash
cp .env.template .env
```

### 3. Применяем миграции alembic

```bash
alembic upgrade head
```

### 4. Запускаем приложение

```bash
python src/main.py
```

Приложение будет доступно по адресу: `http://localhost:8000`

## Вебхук

Эндпоинт `/transactions/webhook` эмулирует обработку платежа от сторонней системы.

### Структура запроса

```json
{
  "transaction_id": "5eae174f-7cd0-472c-bd36-35660f00132b",
  "user_id": 1,
  "account_id": 1,
  "amount": 100,
  "signature": "7b47e41efe564a062029da3367bde8844bea0fb049f894687cee5d57f2858bc8"
}
```

### Формирование подписи

Подпись формируется через SHA256 хеш строки, состоящей из значений объекта в алфавитном порядке ключей и секретного ключа:

```
SHA256({account_id}{amount}{transaction_id}{user_id}{secret_key})
```

### Логика обработки

1. Проверяется подпись запроса
2. Если счёт не существует, он создаётся автоматически
3. Транзакция сохраняется в базе данных (повторная обработка одного `transaction_id` невозможна)
4. Баланс счёта пополняется на сумму транзакции

## Тесты

### Запуск

```bash
pytest
```

### Структура

```
tests/
    conftest.py             # фикстуры: сессия, тестовая БД
    repos/
        test_user.py        # CRUD пользователей
        test_account.py     # CRUD счетов
        test_transaction.py # CRUD транзакций
    services/
        test_auth.py        # аутентификация
        test_webhook.py     # обработка вебхука
    utils/
        test_auth.py        # хеширование паролей, JWT
        test_signature.py   # верификация подписи вебхука
```

### Примечания

- Тесты используют отдельную тестовую БД (SQLite in-memory)
- Каждый тест выполняется в отдельной транзакции с откатом после завершения
- Покрыты репозитории, сервисы со сложной бизнес-логикой и утилиты
